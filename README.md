# JNTU-GV — Official Codebase Report

Overview
- **Purpose:** Frontend codebase for Jawaharlal Nehru Technological University — Gurajada Vizianagaram (JNTU-GV) public website.
- **Status:** Active; production build artifacts are in the `build/` folder and the site is hosted as a static site (Netlify or equivalent).
- **Stack:** React (Create React App structure), JavaScript (ES6+), CSS, Webpack tooling via scripts in `package.json`.

Repository Structure (top-level)
- **public/**: Static HTML and public assets used at build time.
- **build/**: Production-ready, served artifacts (generated by `npm run build`).
- **scripts/**: build helper scripts (e.g., `build-with-timestamp.js`).
- **src/**: Application source code (React components, styles, assets).
- **src/components/**: UI components grouped by feature (HomePage, MainPage, Gallery, Events, etc.).
- **src/assets/**: Media, images, PDFs and other static assets used by the site.
- **package.json**: Project metadata and npm scripts.

Quick Setup (developer)
1. Prerequisites: Node.js LTS (18+ recommended), npm (or yarn).
2. Clone the repo and install:
	```bash
	git clone <repository-url>
	cd JNTU-GV
	npm install
	```
3. Run development server (hot reload):
	```bash
	npm start
	```
4. Create production build:
	```bash
	npm run build
	```

Common Scripts
- `npm start` — Local development server.
- `npm run build` — Build production static files into `build/`.
- `npm run dev` — (if present) alternative dev mode.
- `node scripts/build-with-timestamp.js` — helper used by CI/deploy to tag builds.

Deployment
- Build output lives in `build/` and can be deployed to any static-file host (Netlify, GitHub Pages, AWS S3 + CloudFront, etc.).
- Ensure the web server serves `index.html` for client-side routing (single-page app).

Architecture & Conventions
- Component-driven React structure under `src/components/` grouped by feature.
- Global styles in `src/App.css` and modular per-component styles (e.g., `Homepage.css`).
- Static assets are stored in `src/assets/` and referenced with relative imports.
- Keep components focused: presentational vs container separation is encouraged.

Testing & Quality
- Unit tests (if present) can be run with:
  ```bash
  npm test
  ```
- Linting and formatting should follow the project's established config (ESLint, Prettier) if present — add or run `npm run lint` if configured.

Contribution Guide
- Fork the repository and create feature branches with descriptive names: `feature/<name>` or `fix/<issue>`.
- Open a Pull Request with a clear description and link to any related issue.
- Follow existing code style and component patterns; keep changes focused and small.

Known Issues & Notes
- Build & dev commands have been used in this workspace (e.g., `npm run dev`, `npm run build`); fix local environment or dependency issues if commands fail.
- Large media assets (gallery images) are included in `src/assets/`; consider using an external CDN for performance in future releases.
